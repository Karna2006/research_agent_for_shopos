Checking Logs + Debugging — Free Tools

1. Server logs (best first stop)

Every agent prints to stdout. Watch live:

uvicorn main:app --host 0.0.0.0 --port 8000 --reload 2>&1 | tee shopos.log

What to look for:
[Brand Basics] done (4.2s)             ← OK
[Brand Basics] ERROR (2.1s): timeout   ← broken
[phase2] gather exception: ...         ← parallel agent crash
[synthesis] analyst_brief skipped — ...← LLM rate limit

---
2. Read agent results directly from DB

# Install sqlite browser (free, GUI)
brew install --cask db-browser-for-sqlite

Open shopos.db → Table auditrun → see every agent's JSON column.

Or CLI:
sqlite3 shopos-agent/shopos.db "
SELECT id, url, status, progress_pct, current_agent, error
FROM auditrun ORDER BY id DESC LIMIT 10;"

Check specific agent result:
sqlite3 shopos-agent/shopos.db "
SELECT json_extract(brand_basics, '$.analysis.brand_stage') as stage,
       json_extract(store_cro, '$.pagespeed.mobile_score') as mobile,
       json_extract(geo_visibility, '$.analysis.geo_score') as geo
FROM auditrun WHERE id = 1;"

---
3. Internal agent endpoints — test ONE agent in isolation

Hit each agent individually without running the full pipeline:

curl -X POST http://localhost:8000/internal/agent/geo \
  -H "Content-Type: application/json" \
  -H "X-Internal-Secret: dev-internal-key-change-in-prod" \
  -d '{"url": "https://rarerabbit.in", "brand_name": "Rare Rabbit"}'

If it returns garbage → that specific agent is broken.
If it returns good data → agent is fine, issue is upstream.

---
4. Logfire (free tier — best for FastAPI tracing)

pip install logfire

# Add to main.py after app = FastAPI(...)
import logfire
logfire.configure()
logfire.instrument_fastapi(app)

Free tier: 30 days, full request traces, shows exactly which line failed.
Dashboard: logfire.pydantic.dev

---
5. Sentry (free tier — catches exceptions with stack traces)

pip install sentry-sdk[fastapi]

# Add to main.py top
import sentry_sdk
sentry_sdk.init(dsn="YOUR_FREE_DSN")  # get from sentry.io → free plan

Every unhandled exception → full stack trace in Sentry dashboard. Shows exact line, local variables, request payload.

---
6. Quick agent health check script

Run this anytime to spot broken agents without a full audit:

# Save as test_agents.sh
curl -s http://localhost:8000/health | python3 -m json.tool
curl -s http://localhost:8000/status | python3 -m json.tool
curl -s http://localhost:8000/brands | python3 -c "import sys,json; d=json.load(sys.stdin); print(f'{len(d[\"brands\"])} brands in DB')"

Debugging Decision Tree

Audit stuck / wrong data?
    │
    ├── Check server logs first
    │       "ERROR" or "TIMEOUT" → which agent?
    │
    ├── Hit /internal/agent/{name} for that specific agent
    │       Returns error → agent bug (scraper or LLM prompt)
    │       Returns good data → issue is in orchestrator or synthesis
    │
    ├── Check DB directly
    │       status="failed" + error column has message → exception in pipeline
    │       status="complete" but agent column is NULL → agent silently skipped
    │
    └── Check Groq rate limits
            429 in logs → hit free tier limit (30 req/min)
            Fix: audit one brand at a time, wait 60s between runs
